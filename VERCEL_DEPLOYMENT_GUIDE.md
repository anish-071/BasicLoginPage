# Vercel Deployment Guide for MERN Application

This guide explains how to deploy your full-stack MERN application to Vercel. Vercel allows you to deploy both the frontend (React/Vite) and the backend (Node.js/Express) seamlessly.

## 1. Prepare Backend for Serverless Deployment
Vercel turns Express routes into Serverless Functions. We have already prepared the application with the required configurations:
- `vercel.json` is included in the `backend` folder to route traffic to `server.js`.
- `server.js` exports the Express app to be used by Vercel serverless functions.

## 2. Deploy the Backend
1. Push your local `backend` code and `frontend` code entirely to a GitHub repository.
2. Log in to your [Vercel Dashboard](https://vercel.com/dashboard) and click **Add New Project**.
3. Import your GitHub repository.
4. In the configuration settings:
   - **Project Name**: Choose a name like `my-mern-backend`.
   - **Framework Preset**: Vercel will detect Node.js, leave as is.
   - **Root Directory**: Click Edit, and select the `backend` folder.
   - **Environment Variables**: Add your API secrets:
     - `MONGO_URI`: Your MongoDB Atlas connection string (e.g. `mongodb+srv://...`)
     - `JWT_SECRET`: A secure long random string.
5. Click **Deploy**. Once successfully built, note the live URL of your backend.

## 3. Connect Frontend to the Deployed Backend
Before deploying the React app, you need to point the frontend to the deployed backend link.
1. Open `frontend/src/context/AuthContext.jsx` and `frontend/src/pages/Dashboard.jsx`.
2. Replace `http://localhost:5000` with the actual backend URL generated by Vercel in step 2 (ensure there is no trailing slash).
3. Commit and push these changes to GitHub.

## 4. Deploy the Frontend
1. Back on Vercel Dashboard, click **Add New Project** again.
2. Import the exact same GitHub repository.
3. In the project config:
   - **Project Name**: Choose a name like `my-mern-frontend`.
   - **Framework Preset**: Vercel will auto-detect **Vite**.
   - **Root Directory**: Click Edit, and select the `frontend` folder.
4. Leave the Build Command and Output Directory as default (`npm run build` and `dist`).
5. Click **Deploy**.

Your full-stack Auth app is now perfectly deployed, scalable, and styled with Tailwind CSS!
